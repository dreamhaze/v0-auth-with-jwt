# Отчёт о миграции Variant Generator с React на Nuxt 4

---

## **1. Общий обзор**

- **Цель**: Миграция логики генератора вариантов из React (`adminka/`) в Nuxt 4 (`frontend/`) с разделением на композиблы, утилиты и типы.

---

## **2. Перенесённые элементы**

### **2.1. Типы и утилиты**

| Файл                       | Описание                                                                                        |
| :------------------------- | :---------------------------------------------------------------------------------------------- |
| `types/variant.ts`         | Определены типы для ключей заданий, истории, флагов загрузки и квот.                            |
| `utils/variant/newTest.ts` | Перенесены чистые функции: сортировка отрывков, фильтрация активных элементов, маппинг ответов. |

### **2.2. Композиблы (Логика)**

| Композибл               | Описание                                                                            |
| :---------------------- | :---------------------------------------------------------------------------------- |
| `useVariantState.ts`    | Централизованное состояние генератора (использует `useState` для SSR-безопасности). |
| `useGenerateVariant.ts` | Логика взаимодействия с API: генерация, обновление блоков и отдельных заданий.      |
| `useNavigateScene.ts`   | Логика навигации между сценами (отрывками) произведения.                            |

### **2.3. Компоненты**

| Компонент             | Изменения                                                                               |
| :-------------------- | :-------------------------------------------------------------------------------------- |
| `Create.vue`          | Полностью переписан: логика делегирована в композиблы, код стал чистым и декларативным. |
| `Tasks1.vue`          | Обновлён для поддержки `Set` в качестве хранилища состояния ответов.                    |
| `NewTestTaskList.vue` | Обновлён аналогично `Tasks1.vue`.                                                       |

---

## **3. Проблемы и решения**

| Проблема                                    | Решение                                                                                            |
| :------------------------------------------ | :------------------------------------------------------------------------------------------------- |
| Огромный хук `useNewTestPage` (1500+ строк) | Разбит на логические группы и вынесен в отдельные композиблы (`state`, `generate`, `navigate`).    |
| Зависимости `useMemo` / `useEffect`         | Заменены на `computed` и `watch`, что в Vue/Nuxt работает автоматически без массивов зависимостей. |
| SSR Гидрация                                | Состояние вынесено в `useState`, критические части обёрнуты в `<ClientOnly>`.                      |

---

## **4. Выводы**

Логика генератора успешно мигрирована. Компонент `Create.vue` теперь служит только для оркестрации, а вся бизнес-логика и работа с данными инкапсулирована в композиблах. Это значительно упрощает поддержку и тестирование кода.
