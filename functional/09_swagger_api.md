# Анализ API (Swagger)

## Доступные эндпоинты (публичные)

### 1. Авторизация и пользователи

| Метод    | Роут                 | Описание                        |
| -------- | -------------------- | ------------------------------- |
| **POST** | `/api/auth/login`    | Вход по email/password          |
| **POST** | `/api/auth/register` | Регистрация нового пользователя |
| **POST** | `/api/auth/refresh`  | Обновление токена доступа       |
| **GET**  | `/api/auth/me`       | Данные текущего пользователя    |

**Подписка**: В ответе `/api/auth/me` (после авторизации) есть поля:

```json
{
  "id": "string",
  "email": "string",
  "first_name": "string",
  "last_name": "string",
  "is_blocked": false,
  "is_pro": true // ← поле подписки
}
```

### 2. База знаний (Knowledge Base)

| Метод   | Роут                         | Описание                                  |
| ------- | ---------------------------- | ----------------------------------------- |
| **GET** | `/api/knowledge-base`        | **Публичный!** Получение всей базы знаний |
| **GET** | `/api/knowledge-base/public` | Тоже база знаний                          |
| **GET** | `/api/knowledge-base/export` | Экспорт базы знаний                       |

### 3. Варианты (Variants) — требуют авторизацию 🔒

| Метод    | Роут                             | Описание                  |
| -------- | -------------------------------- | ------------------------- |
| **POST** | `/api/variants/generate`         | **Генерация варианта**    |
| **POST** | `/api/variants/generate/preview` | **Предпросмотр варианта** |

### 4. Административные эндпоинты (только для админов) 🔒

| Метод    | Роут                                                 | Описание                              |
| -------- | ---------------------------------------------------- | ------------------------------------- |
| **PUT**  | `/api/admin/users/{user_id}/subscription/activate`   | **Активация подписки пользователю**   |
| **PUT**  | `/api/admin/users/{user_id}/subscription/deactivate` | **Деактивация подписки пользователю** |
| **GET**  | `/api/admin/knowledge-base/stats`                    | Статистика базы знаний                |
| **POST** | `/api/admin/knowledge-base/import`                   | Импорт базы знаний                    |

## Кешированная база данных

Роут `/api/knowledge-base` публичный и возвращает всю базу знаний — массивный JSON, который стоит рефакторить. Но он рабочий, в условиях сжатых сроков можно его использовать.

## Подписка (Pro статус)

Есть как минимум заготовка системы подписок. Поля:

- `is_pro` в профиле пользователя
- Админские эндпоинты для активации/деактивации подписки

**НО:** Нет публичного/пользовательского эндпоинта для покупки/активации подписки — только админские (для тестирования/управления).

## Потенциальная задача для backend разработчика

**Контекст:**
Приложение — конструктор вариантов ЕГЭ по литературе. Бесплатно доступно ограниченное количество генерации. Есть защищённые роуты для генерации вариантов: `/api/variants/generate`, `/api/variants/generate/preview` (в новой версии генератора используются `/variants/runtime/generate`, возможно есть `/variants/runtime/generate/preview`) Для пользователей с активной подпиской (поле `is_pro`) эти роуты должны быть доступны. Без подписки — недоступны.

### Требуется реализовать

#### 1. Моковый роут для активации подписки

**Роут:** `POST /api/subscription/activate-mock`

- Доступен только авторизованным пользователям (защищён Bearer токеном)
- При вызове активирует подписку на 30 дней для текущего пользователя
- Устанавливает `is_pro = true` и сохраняет `subscription_expires_at` = `now() + 30 days`
- Возвращает обновлённый профиль пользователя

#### 2. Моковый роут для сброса подписки

**Роут:** `POST /api/subscription/reset-mock`

- Доступен только авторизованным пользователям
- Сбрасывает `is_pro = false`, очищает `subscription_expires_at`
- Возвращает обновлённый профиль пользователя

#### 3. Проверка подписки на защищённых роутах

На всех роутах генерации/обновления вариантов:

- `/api/variants/generate`
- `/api/variants/generate/preview`

Добавить middleware/guard, который проверяет:

1. Пользователь авторизован
2. `is_pro === true`
3. `subscription_expires_at > now()` (не истекла)

Если условия не выполнены — возвращать `403 Forbidden` с телом:

```json
{
  "detail": "Pro subscription required. Please activate your subscription."
}
```

#### 4. Обновление `/api/auth/me`

При активной подписке роут уже возвращает `is_pro: true` — это видно в схеме. Дополнительно нужно добавить поле:

```json
{
  "subscription_expires_at": "2024-12-31T23:59:59Z"
}
```
