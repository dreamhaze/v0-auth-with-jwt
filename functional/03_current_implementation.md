# Текущая реализация проекта

## Демо-вариант

- **Ссылка**: [https://ege.kritsky.academy/new_test](https://ege.kritsky.academy/new_test)
- **Что можно увидеть**:
  - Верстка страницы варианта.
  - Формулировки заданий.
  - Логика отображения "кирпичиков" и заданий.

* **Текущий статус бэкенда**

- **Ручка `api/knowledge-base`**: возвращает полный JSON базы знаний.
  - `GET /api/knowledge-base` → возвращает полный JSON базы знаний (~9.5МБ).
  - `PUT /api/knowledge-base` → обновление базы знаний (используется в админке).
- **Ручка `api/variants/runtime/pregenerated`**: возвращает прегенерированный вариант.
- **Изменения**: при сохранении варианта или покупке сборника текст заданий копируется целиком.

* **Фронтенд**:

  - Старая реализация на **React** удалена из папки `./frontend`.
  - Новая реализация пишется на **Nuxt 4** с фокусом на:

    - Серверную предзагрузку данных (Nitro).
    - Кеширование ответов от бэкенда.
    - Оптимизацию рендеринга (избегание блокировки главного потока).

---

## **Ограничения ресурсов**

- **Бэкенд**:

  - Ответы сервера **берём "как есть"** (не меняем логику бэкенда).
  - **Проблема**: Большой объём данных и возможный рассинхрон при обновлении базы через `PUT /knowledge-base`.
    - Пока не можем избавиться от этой архитектуры из-за ограниченных ресурсов бэкендера.

## **Проблемы фронтенда которые решаются сейчас**

1. **Перегруженная ручка `api/knowledge-base`**:

   - Ответ: ~9.5МБ JSON (весь `knowledge_base_state.payload`).
   - **Проблема**: Пересчет на фронтенде (ранее на React) блокировал главный поток.
   - **Решение**: Переносим вычисления на сервер (Nuxt 4 + Nitro).

2. **Генерация вариантов**:

   - Ранее пересчитывалась на фронтенде при каждом рендере (`useMemo` в React).
   - **Проблема**: Долгий рендер и блокировка интерфейса.
   - **Решение**: Серверная предзагрузка и кеширование для роутов:
     - `api/knowledge-base` (кеш инвалидируется при `PUT /knowledge-base`).
     - `api/variants/runtime/pregenerated` (кеш для общего варианта).

3. **Админка**:

   - **Функционал**:
     - Просмотр всех "кирпичиков" (отрывков, стихотворений, заданий).
     - Добавление/редактирование "кирпичиков" через `PUT /knowledge-base`.
   - **Проблема**: Большой объём данных при обновлении базы.
   - **Решение**: Инвалидация кеша для `api/knowledge-base` после `PUT`-запроса. Лоадер при сохранении и блокировка страницы.
