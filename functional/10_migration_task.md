# Задача по миграции проекта с Next.js (React) на Nuxt 4 (Vue)

---

---

## **Контекст**

- В проекте предоставлены **две папки с frontend-ом**:

  - `adminka/` — старая реализация фронтенда на **Next.js (React)**. Содержит **всю вёрстку проекта**, включая публичную часть и админку.
  - `frontend/` — новая реализация на **Nuxt 4 (Vue)**. Содержит только заготовку публичной части.

- **Цель**:
  1. Перенести вёрстку из `adminka/` в `frontend/` (Nuxt 4).
  2. Разделить функционал: оставить админку в `adminka/`, а публичную часть — в `frontend/`.
  3. Задокументировать процесс миграции.

---

---

---

## **Задача 1: Перенос вёрстки из Next.js в Nuxt 4**

### **Цель**

Перенести **всю вёрстку** (стили, компоненты, шаблоны) из `adminka/` в `frontend/` с сохранением визуального сходства.

---

### **Список страниц для переноса**

Перенести вёрстку **всех** перечисленных страниц (включая стили, компоненты и расстановку элементов):

- `/` (home page)
- `/about/`
- `/author-variant/`
- `/cart/`
- `/feedback/`
- `/login/`
- `/new_test/`
- `/policy/`
- `/privacy/`
- `/profile/*`
- `/rules/`
- `/shop/*`
- `/terms/`

---

### **Требования к переносу**

1. **Стили и дизайн**:

   - Перенести **основные стили**:
     - Шрифты (`font-family`, размеры, веса).
     - Цвета (акцентный, основной, фоновый и др.).
     - Отступы, границы, тени и другие визуальные параметры.
   - Использовать **CSS/SCSS модули** или **Tailwind CSS** (если уже подключён в `frontend/`).
   - Если в `adminka/` используются **глобальные стили** (например, `globals.css`), перенести их в `frontend/assets/css/main.css` или аналогичный файл.

2. **Компоненты**:

   - Выделить **общие компоненты** (например, `Header`, `Footer`, `Container`, `Button`, `Card`) и перенести их в папку `frontend/components/`.
   - Разбить страницы на **подкомпоненты** (например, `HeroSection`, `TaskList`, `Sidebar`).
   - Использовать **композиционный API** (Vue 3) для логики компонентов.

3. **Шаблоны**:

   - Перенести **основной шаблон** (layout) из `adminka/` в `frontend/layouts/default.vue`.
   - Убедиться, что **header**, **footer** и **container** отображаются на всех страницах.

4. **Адаптивность**:

   - Проверить, что вёрстка корректно отображается на **мобильных устройствах** (если это было реализовано в `adminka/`).

5. **Иконки и статические файлы**:
   - Перенести **иконки**, **изображения** и другие статические файлы в `frontend/public/`.
   - Обновить пути к файлам в коде (например, `/images/logo.png` → `/logo.png`).

---

### **Примеры переноса**

#### **1. Перенос глобальных стилей**

- **Исходник** (`adminka/styles/globals.css`):
  ```css
  :root {
    --primary-color: #3498db;
    --accent-color: #e74c3c;
    --font-main: 'Roboto', sans-serif;
  }
  body {
    font-family: var(--font-main);
    margin: 0;
    padding: 0;
  }
  ```
- **Результат** (`frontend/assets/css/main.css`):
  ```css
  :root {
    --primary-color: #3498db;
    --accent-color: #e74c3c;
    --font-main: 'Roboto', sans-serif;
  }
  body {
    font-family: var(--font-main);
    margin: 0;
    padding: 0;
  }
  ```

#### **2. Перенос компонента `Header`**

- **Исходник** (`adminka/components/Header.jsx`):
  ```jsx
  export default function Header() {
    return (
      <header className="header">
        <div className="container">
          <a href="/" className="logo">
            Logo
          </a>
          <nav className="nav">
            <a href="/about">About</a>
            <a href="/login">Login</a>
          </nav>
        </div>
      </header>
    );
  }
  ```
- **Результат** (`frontend/components/Header.vue`):

  ```vue
  <template>
    <header class="header">
      <div class="container">
        <NuxtLink to="/" class="logo">Logo</NuxtLink>
        <nav class="nav">
          <NuxtLink to="/about">About</NuxtLink>
          <NuxtLink to="/login">Login</NuxtLink>
        </nav>
      </div>
    </header>
  </template>

  <style scoped>
  .header {
    background: var(--primary-color);
    padding: 1rem 0;
  }
  .logo {
    color: white;
    font-weight: bold;
  }
  </style>
  ```

#### **3. Перенос страницы `/new_test/`**

- **Исходник** (`adminka/pages/new_test.jsx`):

## **Задача 2: Разделение функционала между проектами**

### **Цель**

- Оставить в `adminka/` **только админский функционал** (роут `/admin`).
- Удалить из `frontend/` **весь функционал, связанный с `/admin`**.
- Перенести админский функционал из `adminka/` в корень приложения `frontend/` (если он там отсутствует).

---

### **Шаги**

#### **1. Очистка `adminka/`**

- Удалить **все файлы и папки**, не связанные с роутом `/admin`:
  - Удалить страницы: `/`, `/about/`, `/login/` и др. (оставить только `/admin`).
  - Удалить компоненты, которые используются **только в публичной части** (например, `Header`, `Footer` для публичных страниц).
  - Оставить **только**:
    - Страницы и компоненты, связанные с `/admin`.
    - Логику авторизации для админки.
    - Стили, специфичные для админки.

#### **2. Очистка `frontend/`**

- Удалить **весь функционал, связанный с `/admin`**:
  - Удалить страницы `/admin` и все подстраницы (например, `/admin/dashboard`).
  - Удалить компоненты, которые используются **только в админке**.
  - Удалить роуты, связанные с админкой (в `nuxt.config.ts` или `app.vue`).

#### **3. Перенос админского функционала в `frontend/`**

- Если в `adminka/` есть **уникальный функционал админки**, который отсутствует в `frontend/`:
  - Перенести его в `frontend/` (например, в папку `frontend/pages/admin/`).
  - Обновить роуты в `frontend/` для поддержки `/admin`.
