# Задача для агента

## Цель

Переработать фронтенд на **Nuxt 4** с учетом серверной предзагрузки и кеширования для роутов:

1. `api/knowledge-base`
2. `api/variants/runtime/pregenerated`

## Страницы для реализации

1. **Главная** (`/`):
   - Отображать количество возможных вариантов (на основе `knowledge-base`).
   - Кнопка "Сгенерировать вариант" → ведет на `/public-variant` (можно придумать более лаконичное название).
2. **Страница прегенерированного варианта** (`/public-variant`):
   - Получать данные из `/api/variants/runtime/pregenerated`.
   - Отображать вариант в формате ЕГЭ (см. демо).
3. **Админка** (`/admin`):
   - Отображать все "кирпичики" (отрывки, стихотворения, задания) из `knowledge-base`.
   - Кнопка "Обновить базу" → инвалидировать кеш.

## Технические требования

- **Серверная логика**:
  - Кешировать ответы для `knowledge-base` и `pregenerated`.
  - Инвалидировать кеш при изменениях в админке.
- **Фронтенд**:
  - Не блокировать рендер (использовать `useAsyncData` или аналоги).
  - При долгих вычислениях — лоадер.
- **Fallback**:
  - Если сервер не справляется — использовать Web Worker для вычислений на клиенте.

## Ограничения

- **Не менять бэкенд**: только анализировать payloads.
- **Не тратить время на стилизацию**: только функционал.
- **Использовать существующий код** в папке `./frontend` как основу.

## Результат

- Готовый проект в Docker-контейнере.
- Критические тесты на получение HTML-страниц с предзагруженными данными.
- Отчет о прохождении тестов.
