# Ограничения и лучшие практики

## Ограничения

1. **Не менять бэкенд**:
   - Только анализировать payloads для соответствия фронтенда.
2. **Минимализм**:
   - Не реализовывать стилизацию (использовать базовые стили Nuxt).
   - Не добавлять лишние фичи (только то, что указано в задаче).
3. **Производительность**:
   - Кешировать все тяжелые вычисления.
   - Не блокировать главный поток.
4. **Стек**:
   - Только **Nuxt 4** (гибридный режим).
   - Для state-менеджмента: `useState` или `pinia` (если необходимо).
   - Для HTTP-запросов: `$fetch` или `useFetch`.

## Лучшие практики

1. **Кеширование**:
   - Использовать `useStorage()` (Nitro) для серверного кеша.
   - Указывать TTL (time-to-live) для кеша.
2. **Асинхронность**:
   - Использовать `useAsyncData` для загрузки данных на сервере.
   - Для клиентских вычислений — `useAsyncData` + `onMounted`.
3. **Обработка ошибок**:
   - Показывать пользователю понятные сообщения при ошибках.
   - Логировать ошибки в консоль (для отладки).
4. **Тестирование**:
   - Проверять, что HTML-страницы содержат предзагруженные данные.
   - Тестировать инвалидацию кеша.
