# Юзкейс пользователя: Сценарий работы учителя/репетитора

---

## **Целевой посетитель**

- **Кто**: Учитель или репетитор по литературе (включая владельца сайта).
- **Цель**: Использовать конструктор для генерации уникальных вариантов ЕГЭ, соответствующих формату, и проверки их качества перед работой с учениками.

---

## **Первое посещение: Ознакомление с системой**

### **Шаг 1: Просмотр демо-варианта**

- Пользователь заходит на **главную страницу** (`/`).
- Видит **кнопку "Посмотреть демо-вариант"** (ведет на `/new_test`).
- Система отображает **прегенерированный вариант** (роут `/api/variants/runtime/pregenerated`).
  - Вариант на 100% соответствует **формату ЕГЭ** (структура, типы заданий, формулировки).
  - Отличие от бумажных сборников: **интерактивность** и **динамическая генерация**.

### **Шаг 2: Проверка соответствия формату ЕГЭ**

- Учитель проверяет:
  - Соответствие структуры варианта **официальным требованиям ЕГЭ** (части 1 и 2, типы заданий, формулировки).
  - Наличие всех необходимых элементов:
    - Отрывки из произведений.
    - Стихотворения.
    - Задания с кратким и развернутым ответом.
  - Отсутствие ошибок в формулировках и ответах.

---

## **Работа с конструктором**

### **Шаг 3: Генерация кастомного варианта**

- Пользователь переходит в **конструктор** (например, `/constructor`).
- Видит опции для генерации:
  1. **Случайный вариант**:
     - Кнопка **"Сгенерировать случайный вариант"** → система собирает вариант из случайных "кирпичиков" (отрывков, стихотворений, заданий).
  2. **Вариант по критериям**:
     - Выбор **автора** (например, Пушкин, Лермонтов).
     - Выбор **произведения** (например, "Евгений Онегин", "Парус").
     - Кнопка **"Сгенерировать по фильтру"** → система собирает вариант на основе выбранных критериев.
  3. **Обновление отдельного задания**:
     - Кнопка **"Обновить задание"** рядом с каждым заданием → система заменяет **только это задание**, сохраняя остальные.

### **Шаг 4: Ограничения бесплатного доступа**

- **Лимит на генерацию**:
  - Бесплатно можно обновить **не более 3 раз** (любые комбинации: генерация новых вариантов или обновление отдельных заданий).
- **Блокировка после исчерпания лимита**:
  - При попытке сгенерировать 4-й вариант или обновить 4-е задание → система показывает **модальное окно с предложением купить подписку**.
  - Текст: _"Вы исчерпали бесплатные генерации. Подписка на месяц даёт неограниченное количество генераций и 3 скачивания в день."_

---

## **Подписка: Расширенные возможности**

### **Шаг 5: Покупка подписки**

- Пользователь оформляет **месячную подписку**.
- Получает доступ к:
  - **Неограниченной генерации** вариантов и обновлению заданий.
  - **Скачиванию вариантов** (до **3 скачиваний в день** в формате PDF).
    - Ограничение на скачивания введено, чтобы предотвратить массовую выгрузку базы знаний.

### **Шаг 6: Работа с учениками**

- Учитель может:
  - Генерировать **индивидуальные варианты** для каждого ученика.
  - Давать задания на **повторение конкретных тем** (например, обновлять только задания по лирике Лермонтова).
  - Скачивать варианты для **печати или отправки ученикам** (в пределах лимита скачиваний).

---

## **Ключевые требования к системе**

### **Функциональные**

1. **Генерация вариантов**:

   - Соответствие **формату ЕГЭ** (структура, типы заданий, формулировки).
   - Возможность генерации **по фильтрам** (автор, произведение).
   - Возможность обновления **отдельных заданий**.

2. **Ограничения**:

   - Лимит на **3 бесплатные генерации/обновления**.
   - Лимит на **3 скачивания в день** для подписчиков.

3. **Интерактивность**:
   - Мгновенное обновление страницы при генерации/обновлении.
   - Кеширование сгенерированных вариантов для ускорения загрузки.

### **Технические**

- **Скорость**:
  - Генерация варианта должна занимать **< 2 секунды** (включая рендер).
  - При долгих вычислениях (например, при обновлении базы) — показывать **лоадер**.
- **Кеширование**:
  - Прегенерированный вариант (`/pregenerated`) кешируется для всех пользователей.
  - База знаний (`/knowledge-base`) кешируется с **инвалидацией при изменениях** (через админку).

### **UX/UI**

- **Минимализм**:
  - Интерфейс должен быть **интуитивно понятным** для учителей (без лишних элементов).
  - Кнопки генерации и обновления должны быть **хорошо видны** и доступны.
- **Обратная связь**:
  - При достижении лимита — **четкое объяснение** (например, модальное окно с предложением подписки).
  - При генерации — **визуальная индикация** (лоадер, уведомление об успешной генерации).
